## Overview

Safaris Web provides multiple search and filtering methods to help you find the perfect tour. This guide covers the search functionality, category filtering, subcategory navigation, and tips for effective tour discovery.

## Search Functionality

### Main Search Feature

**Route:** `/search`

**Controller:** `HomeController::search(Request $request)`

**How It Works:**

The search system uses Laravel's query builder to find tours matching your search term:

```php
Tours::where([
    ['hotel', '!=', Null],
    [function ($query) use ($request) {
        if(($term = $request->item)) {
            $query->orWhere('hotel', 'LIKE', '%' . $term . '%')->get();
        }
    }]
])->orderBy("id", "desc")->paginate(10)
```

<Info>
  Search primarily matches against the `hotel` field but can be expanded to include other tour attributes.
</Info>

### Using Search

<Steps>
  <Step title="Access Search">
    Navigate to `/search` or use the search box on any page
  </Step>
  <Step title="Enter Search Term">
    Type your search query in the `item` parameter. Examples:
    - Hotel names: "Serena", "Sarova", "Mara Sopa"
    - Destinations: "Masai Mara", "Amboseli"
    - Location keywords
  </Step>
  <Step title="View Results">
    Search returns paginated results (10 per page) ordered by most recent first
  </Step>
  <Step title="Refine Search">
    Modify your search term to narrow or broaden results
  </Step>
</Steps>

</Tip>

## Category Filtering

### Understanding Categories

Categories are managed in the `categories` table with structure:

| Field | Type | Description |
|-------|------|-------------|
| `id` | Integer | Unique identifier |
| `category_name` | Text | Main category name |
| `sub_category` | Text | First subcategory |
| `sub_category1` | Text | Second subcategory |
| `sub_category2` | Text | Third subcategory |
| `created_at` | Timestamp | Creation date |
| `updated_at` | Timestamp | Last update |

<Info>
  Each category can have up to 3 subcategories for granular filtering.
</Info>

### Main Category Filtering

**Route:** `/tours/{category}`

**Controller:** `HomeController::tours($category)`

**How It Works:**

<Steps>
  <Step title="Select Category">
    Click on any category link or navigate to `/tours/{category}` where `{category}` is the category name
    
    **Common categories:**
    - Wildlife Safaris
    - Beach Holidays
    - Cultural Tours
    - Adventure Tours
    - Mountain Climbing
    - Bird Watching
    - Photography Safaris
  </Step>
  <Step title="View Filtered Tours">
    All tours in that category are loaded:
    ```php
    Tours::where('category', $category)->get()
    ```
  </Step>
  <Step title="See Category Info">
    The latest tour in the category provides additional context:
    ```php
    Tours::where('category', $category)->latest()->first()
    ```
    This displays the category's `additional_info` field
  </Step>
</Steps>

**Page Displays:**
- All tours matching the selected category
- Category-specific information from latest tour
- All available categories for easy switching
- 2 latest blog posts

### Subcategory Filtering

**Route:** `/tours/categories/{sub_category}`

**Controller:** `HomeController::tour_cat($sub_category)`

**More Specific Filtering:**

<Steps>
  <Step title="Navigate to Subcategory">
    From a category page, click on subcategory links
  </Step>
  <Step title="View Subcategory Tours">
    Tours filtered by subcategory:
    ```php
    Tours::where('sub_category', $sub_category)->get()
    ```
  </Step>
  <Step title="Context Information">
    Latest tour provides subcategory-specific info:
    ```php
    Tours::where('sub_category', $sub_category)->latest()->first()
    ```
  </Step>
</Steps>

**Example Hierarchy:**
- **Category:** Wildlife Safaris
  - **Subcategory:** Big Five Safaris
  - **Subcategory:** Bird Watching Safaris
  - **Subcategory:** Migration Safaris

<Accordion title="Why Two Filtering Methods?">
  **Category filtering** (`/tours/{category}`) provides broad groupings for general browsing.
  
  **Subcategory filtering** (`/tours/categories/{sub_category}`) offers specific niches within main categories for targeted searches.
</Accordion>

## Tour Data Searchable Fields

Tours contain multiple fields that could match your search:

<Tabs>
  <Tab title="Currently Searchable">
    **Hotel Field**
    - Primary search field
    - Contains accommodation name
    - Examples: "Sarova Mara Game Camp", "Serena Lodge"
    
    **Search Implementation:**
    ```php
    $query->orWhere('hotel', 'LIKE', '%' . $term . '%')
    ```
  </Tab>
  <Tab title="Filterable Fields">
    **Category**
    - Main category classification
    - Route: `/tours/{category}`
    
    **Sub_category**
    - Specific subcategory
    - Route: `/tours/categories/{sub_category}`
    
    **Location**
    - Geographic destination
    - Not directly searchable but visible in results
  </Tab>
  <Tab title="Other Tour Data">
    While not directly searchable, these fields are displayed in results:
    - Transport type
    - Meal plans
    - Pricing (per person sharing, single room)
    - Additional information
    - Images (up to 6 per tour)
  </Tab>
</Tabs>

<Tip>
  When searching, focus on hotel/lodge names for best results. Use category filters for broader discovery.
</Tip>

## Advanced Filtering Strategies

### Combination Approach

<Steps>
  <Step title="Start with Category">
    Browse by main category to see all tours in your area of interest
    
    Example: `/tours/Wildlife Safaris`
  </Step>
  <Step title="Narrow with Subcategory">
    Use subcategory to get more specific
    
    Example: `/tours/categories/Big Five Safaris`
  </Step>
  <Step title="Search Within Results">
    If you know specific hotels or destinations, use search
    
    Example: Search "Masai Mara" within Big Five Safaris
  </Step>
  <Step title="Compare Options">
    Review pricing, transport, and meal options in results
  </Step>
</Steps>

### Filtering by Location

**Approach:**

1. **Use Search:** Enter location name in search (e.g., "Amboseli")
2. **Check Tour Names:** Hotels often include location (e.g., "Amboseli Serena Safari Lodge")
3. **Browse Categories:** Location-based categories (e.g., "Coastal Tours", "Rift Valley")
4. **Read Descriptions:** Location field displayed in tour details

<Info>
  Tours store location in the `location` field (text type), which includes destination information.
</Info>

## Search Limitations

<Warning>
  **Current Limitations:**
  - Search primarily matches hotel field only
  - No multi-field search (e.g., location + category)
  - No price range filtering
  - No date-based filtering
  - No advanced search operators (AND, OR, NOT)
</Warning>

**Workarounds:**
- Use category + subcategory filtering for multi-criteria search
- Manually compare prices in results
- Check `additional_info` field for detailed inclusions
