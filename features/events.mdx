## Overview

Safaris Web features a dedicated events section showcasing special experiences, cultural events, seasonal attractions, and time-specific activities across Kenya. Events complement tours and accommodations by offering unique, scheduled experiences.

## Event Structure

Events are managed through the `Event` model (`app/Models/Event.php`) and stored in the `events` database table.

## Browsing Events

### Main Events Page

**Route:** `/events`

**Controller:** `EventController::index()`

**Display Logic:**
```php
$events = Event::orderBy('id','desc')->get();
```

**Features:**
- All published events
- Ordered by most recent first
- Shows upcoming and current events
- Category filtering available

### Homepage Display

Events also appear on the homepage with date filtering:

**Route:** `/`

**Controller:** `HomeController::index()`

**Query:**
```php
$date = Carbon::today();
$events = Event::orderBy('id','desc')->whereDate("when", "=>", $date);
```

<Info>
  Homepage only shows current and future events (where event date >= today).
</Info>

### Unique Features

<AccordionGroup>
  <Accordion title="Time-Specific">
    Events are unique in having specific date and time fields:
    - `when` - The event date
    - `starts` - Start time
    - `ends` - End time
    
    This distinguishes events from always-available tours and accommodations.
  </Accordion>
  <Accordion title="Slug-Based URLs">

  <Accordion title="Date Filtering">

  </Accordion>
</AccordionGroup>

## Admin Management Routes

Authenticated admins manage events through these routes:

<Tabs>
  <Tab title="View All">
    **Admin View:**
    - Route: `/all-events`
    - Controller: `EventController::all()`
    - Middleware: `auth`, `verified`
    - Shows all events including unpublished and past events
  </Tab>
  <Tab title="Create">
    **New Event:**
    - Form: `/event/new` (GET)
    - Submit: `/event/create` (POST)
    - Middleware: `auth`, `verified`
    - Requires: name, category, location, when, starts, ends, description, price, 2 images
  </Tab>
  <Tab title="Edit">
    **Update Event:**
    - Form: `/event/edit/{id}` (GET)
    - Update: `/event/update/{id}` (PUT)
    - Middleware: `auth`, `verified`
    - Can update all fields including dates/times
  </Tab>
  <Tab title="Publish">
    **Toggle Status:**
    - Route: `/event/publish/{id}` (PUT)
    - Toggles `published` between 0 and 1
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Delete">
    **Remove Event:**
    - Route: `/event/delete/{id}` (DELETE)
    - Permanently removes event
    - Middleware: `auth`, `verified`
  </Tab>
</Tabs>

<Warning>
  All admin routes require authentication and email verification. Regular users cannot create or manage events.
</Warning>

## Event Creation & Publishing

<Steps>
  <Step title="Create Event">
    Admin creates event with all required details:
    - Name, category, location
    - Date (when), start time, end time
    - Description and price
    - 2 images
    - Auto-generates slug from name
  </Step>
  <Step title="Draft Status">
    Initially created with `published = 0`
  </Step>
  <Step title="Review">
    Admin verifies:
    - Date and time accuracy
    - Description completeness
    - Pricing clarity
    - Image quality
  </Step>
  <Step title="Publish">
    Set `published = 1` to make visible to public
  </Step>
  <Step title="Monitor">
    Event appears on `/events` page and homepage (if upcoming)
  </Step>
</Steps>

## Event Scheduling Details

### Date Field (`when`)

**Format:** Date (YYYY-MM-DD)

**Usage:**
- Determines if event is upcoming or past
- Homepage filtering: `whereDate("when", "=>", $date)`
- Displayed to users for planning

**Examples:**
- `2026-07-15` - July 15, 2026
- `2026-12-31` - New Year's Eve event

### Time Fields

**Starts Field:**
- Format: Time (HH:MM:SS)
- Event start time
- Helps attendees plan arrival

**Ends Field:**
- Format: Time (HH:MM:SS)
- Event end time
- Duration planning

**Example:**
- Starts: `09:00:00` (9:00 AM)
- Ends: `17:00:00` (5:00 PM)
- Duration: 8 hours

<Info>
  Time fields use 24-hour format in database but can be displayed in 12-hour format with AM/PM.
</Info>

## Events vs Other Features

<Tabs>
  <Tab title="vs Tours">
    **Events:**
    - Specific dates and times
    - Single activity or experience
    - Price per person/ticket
    - Shorter duration (hours/single day)
    
    **Tours:**
    - Flexible dates
    - Complete multi-day packages
    - Includes accommodation, transport, meals
    - Multiple days
  </Tab>
  <Tab title="vs Accommodations">
    **Events:**
    - Time-specific activities
    - Scheduled experiences
    - Typically don't include lodging
    
    **Accommodations:**
    - Always available (subject to booking)
    - No specific event times
    - Focus on lodging
  </Tab>
  <Tab title="vs Services">
    **Events:**
    - Scheduled for specific dates/times
    - Group activities often
    - Event-based
    
    **Services:**
    - On-demand availability
    - Individual services
    - Service-based
  </Tab>
</Tabs>

<Tip>
  **Combine Features:** Book a tour package, add accommodation near an event location, and book the event for a complete, customized experience.
</Tip>
