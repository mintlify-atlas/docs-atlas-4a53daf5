## Overview

Safaris Web features a dedicated accommodations section separate from full safari packages. Browse and book hotels, lodges, camps, and resorts across Kenya through the accommodations feature.

<Info>
  Accommodations are managed through the `Accomodation` model (`app/Models/Accomodation.php`) - note the spelling in the codebase.
</Info>

## Browsing Accommodations

### Main Accommodations Page

**Route:** `/accomodations`

**Controller:** `AccomodationController::index()`

**Display:**

**Features:**
- All published accommodations
- Ordered by most recent first
- Category filter options
- Image galleries for each property

## Accommodation Model

### Fillable Fields

From `app/Models/Accomodation.php`:

```php
protected $fillable = [
    "name", 
    "location", 
    "description", 
    "category", 
    "rates"
];
```

**Why These Fields Are Fillable:**
- Allows mass assignment during creation/update
- Core fields that admins need to set
- Security: Other fields (user_id, published) controlled separately

### Slug Generation

The `slug` field provides:
- Unique URL-friendly identifiers
- SEO benefits
- Clean, readable URLs

**Example:**
- Name: "Serena Safari Lodge Masai Mara"
- Slug: `serena-safari-lodge-masai-mara`
- URL: `/accomodation/serena-safari-lodge-masai-mara`

<Warning>
  Slugs must be unique (unique constraint in database). System prevents duplicate slugs.
</Warning>

## Admin Management Routes

Authenticated admins can manage accommodations:

### All Management Routes

<Tabs>
  <Tab title="View All">
    **Admin View:**
    - Route: `/all-accomodations`
    - Controller: `AccomodationController::all()`
    - Middleware: `auth`, `verified`
    - Shows all accommodations including unpublished
  </Tab>
  <Tab title="Create">
    **New Accommodation:**
    - Form: `/accomodation/new` (GET)
    - Submit: `/accomodation/create` (POST)
    - Middleware: `auth`, `verified`
    - Requires: name, location, description, category, rates, 4 images
  </Tab>
  <Tab title="Edit">
    **Update Accommodation:**
    - Form: `/accomodation/edit/{id}` (GET)
    - Update: `/accomodation/update/{id}` (PUT)
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Publish">
    **Toggle Status:**
    - Route: `/accomodation/publish/{id}` (PUT)
    - Toggles `published` field between 0 and 1
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Delete">
    **Remove:**
    - Route: `/accomodation/delete/{id}` (DELETE)
    - Permanently deletes accommodation
    - Middleware: `auth`, `verified`
  </Tab>
</Tabs>

<Info>
  All admin routes require authentication and email verification for security.
</Info>

## Publishing Workflow

<Steps>
  <Step title="Create Entry">
    Admin creates accommodation with all details and 4 images
  </Step>
  <Step title="Draft Status">
    Initially created with `published = 0` (not visible to public)
  </Step>
  <Step title="Review">
    Admin reviews information, images, rates for accuracy
  </Step>
  <Step title="Publish">
    Admin sets `published = 1` via publish route
  </Step>
  <Step title="Live">
    Accommodation now visible on `/accomodations` page and homepage
  </Step>
</Steps>

**Unpublishing:**
- Doesn't delete the accommodation
- Hides from public view
- Allows edits before republishing
- Useful for seasonal properties or during renovations

## Accommodation vs Tour Accommodations

### Key Differences

<AccordionGroup>
  <Accordion title="Standalone Accommodations">
    **Route:** `/accomodations`
    
    **Purpose:**
    - Book accommodation only
    - No transport or full package included
    - Direct hotel/lodge booking
    - Flexible for independent travelers
    
    **Model:** `Accomodation`
    
    **Fields:** name, location, description, category, rates, 4 images
  </Accordion>
  <Accordion title="Tour Accommodations">
    **Included in:** Tours/Safaris packages
    
    **Purpose:**
    - Part of complete safari package
    - Includes transport, meals, activities
    - Hotel specified in tour's `hotel` field
    - Package pricing
    
    **Model:** `Tours`
    
    **Fields:** hotel field contains accommodation name as part of tour
  </Accordion>
</AccordionGroup>

<Tip>
  **Choose Standalone Accommodations if:**
  - You want accommodation only
  - You're arranging your own transport
  - You prefer flexible, independent travel
  
  **Choose Tour Package if:**
  - You want all-inclusive safari experience
  - You need transport and guide services
  - You prefer organized itineraries
</Tip>

## Images & Visual Content

**Four Required Images:**

<Steps>
  <Step title="Image 1 - Exterior">
    Main property view, entrance, or iconic exterior shot
  </Step>
  <Step title="Image 2 - Rooms">
    Interior room view showing accommodation quality
  </Step>
  <Step title="Image 3 - Facilities">
    Key amenities like restaurant, pool, spa, lounge
  </Step>
  <Step title="Image 4 - Surroundings">
    Views, grounds, location context, or additional facilities
  </Step>

**Technical Details:**
- Storage: Laravel public storage (`storage/images`)
- Fields: `image`, `image2`, `image3`, `image4`
- All four images required (validation enforced)

<Warning>
  Unlike tours (which support up to 6 images), accommodations require exactly 4 images.
</Warning>
