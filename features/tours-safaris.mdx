## Overview

Safaris Web offers a comprehensive collection of Kenya safari tours and holiday packages. This page explains tour categories, pricing structures, tour details, and how tours are organized on the platform.

## Tour Structure

Tours are managed through the `Tours` model (`app/Models/Tours.php`) and stored in the `tours` database table.

From the `create_tours_table` migration:

## Pricing Structure

### Dual Pricing System

All tours offer two pricing options:

<Tabs>
  <Tab title="Per Person Sharing">
    **Field:** `per_person_sharing`
    
    **What It Means:**
    - Cost per person when sharing a double/twin room
    - Most economical option
    - Requires minimum 2 travelers
    - Common for couples, friends, family members
    
    **Typical Inclusions:**
    - Shared accommodation (double/twin room)
    - All standard tour inclusions
    - Transport, meals, activities as specified
    
    <Tip>
      Per person sharing rates typically offer 30-40% savings compared to single room rates
    </Tip>
  </Tab>
  <Tab title="Single Room">
    **Field:** `single_room`
    
    **What It Means:**
    - Cost for solo traveler with private room
    - Premium pricing (single supplement applied)
    - Complete privacy
    - No roommate required
    
    **Typical Inclusions:**
    - Private room accommodation
    - All standard tour inclusions
    - Transport, meals, activities as specified
    
    <Note>
      Single room supplements cover the cost of allocating an entire room to one person
    </Note>
  </Tab>
</Tabs>

## Tour Details

### Accommodation Information

**Hotel Field:**

The `hotel` field contains the primary accommodation name:

- Luxury lodges (e.g., "Serena Safari Lodge")
- Tented camps (e.g., "Mara Sopa Camp")
- Beach resorts (e.g., "Diani Reef Beach Resort")
- Budget camps
- Mid-range hotels

**Accommodation Standards:**
- Verified properties
- Location-appropriate lodging
- Quality matches tour category

### Transport Details

**Transport Field:**

Contains detailed transportation information:

<Tabs>
  <Tab title="Vehicle Types">
    - **4x4 Safari Vehicles**: Standard for wildlife safaris, pop-up roof for game viewing
    - **Mini-vans**: Budget-friendly option for smaller groups
    - **Luxury Land Cruisers**: Premium safari experience
    - **Coaches**: For larger groups or specific tours
    - **Flight Transfers**: For remote destinations
  </Tab>
  <Tab title="Transport Inclusions">
    Typically includes:
    - Airport/hotel pickup and drop-off
    - All inter-destination transfers
    - Game drive vehicles with experienced drivers
    - Fuel and vehicle maintenance
    - Driver/guide services
  </Tab>
</Tabs>

### Meal Plans

**Meals Field:**

Specifies what meals are included:

<AccordionGroup>
  <Accordion title="Full Board (FB)">
    - Breakfast
    - Lunch
    - Dinner
    - Usually excludes drinks
    
    **Best for:** Multi-day safaris, remote locations
  </Accordion>
  <Accordion title="Half Board (HB)">
    - Breakfast
    - Dinner
    - Lunch at your own cost
    
    **Best for:** Beach holidays, city tours with exploration time
  </Accordion>
  <Accordion title="All Inclusive (AI)">
    - All meals
    - Snacks
    - Most drinks (alcoholic and non-alcoholic)
    - Some activities
    
    **Best for:** Resort stays, luxury packages
  </Accordion>
  <Accordion title="Bed & Breakfast (B&B)">
    - Breakfast only
    - Lunch and dinner at own cost
    
    **Best for:** City tours, flexible itineraries
  </Accordion>
</AccordionGroup>

### Additional Information

**Additional_Info Field** (nullable text):

This optional field contains:

- Detailed itineraries (day-by-day schedules)
- Inclusions (what's covered in the price)
- Exclusions (what's not included)
- Special notes (visa requirements, vaccination info)
- Best time to visit
- Physical fitness requirements
- Age restrictions
- Group size information

<Tip>
  Always read the additional_info field thoroughly before booking. It contains crucial details about what to expect.
</Tip>

## Tour Images

### Image Storage

**Storage Location:** `storage/images` directory (Laravel public storage)

**Image Fields:**
- `image` - Required primary image
- `image1` - Required secondary image
- `image2` through `image6` - Optional additional images

**Accepted Formats:** JPG, JPEG, PNG, SVG, WebP

**Validation Rule:**
```php
'image' => 'required|mimes:jpg,jpeg,svg,png,webp'
```

## Tour Management Routes

### Admin Routes (Authentication Required)

<Tabs>
  <Tab title="Viewing">
    **All Tours:**
    - Route: `/addtour`
    - Controller: `ToursController::index()`
    - Display: `Tours::latest()->paginate(20)`
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Creating">
    **New Tour Form:**
    - Route: `/addtour/create` (GET)
    - Submit: `/addtour/store` (POST)
    - Middleware: `auth`, `verified`
    - Requires all fields including 2 images minimum
  </Tab>
  <Tab title="Editing">
    **Edit Tour:**
    - Route: `/addtour/edit/{id}` (GET)
    - Update: `/addtour/update/{id}` (POST)
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Publishing">
    **Toggle Published Status:**
    - Route: `/addtour/publish/{id}` (PUT)
    - Toggles between 0 (unpublished) and 1 (published)
    - Middleware: `auth`, `verified`
  </Tab>
</Tabs>

<Warning>
  These admin routes require authentication and email verification. Regular users cannot create or edit tours.
</Warning>

## Tour Creation Validation

When creating tours, the following validation applies:

```php
$request->validate([
    'hotel' => 'required',
    'transport' => 'required',
    'image' => 'required|mimes:jpg,jpeg,svg,png,webp',
    'image1' => 'required|mimes:jpg,jpeg,svg,png,webp',
    'single_room' => 'required',
    'per_person_sharing' => 'required',
    'meals' => 'required',
    'category' => 'required',
    'sub_category' => 'required',
    'location' => 'required',
]);
```

**Required Fields:** hotel, transport, 2 images, both pricing options, meals, category, subcategory, location

**Optional Fields:** image2-6, additional_info

## Publishing Workflow

<Steps>
  <Step title="Tour Creation">
    Admin creates tour with all required information via `/addtour/create`
  </Step>
  <Step title="Initial Status">
    Tour created with `published = 0` (not visible to users)
  </Step>
  <Step title="Review">
    Admin reviews tour details, images, pricing
  </Step>
  <Step title="Publish">
    Admin toggles published status to 1 via `/addtour/publish/{id}`
  </Step>
  <Step title="Live">
    Tour now appears on `/safaris` page and in category listings
  </Step>
</Steps>

**Toggle Logic:**
```php
if ($tour->published == 0) {
    $tour->published = 1;
} else {
    $tour->published = 0;
}
$tour->update();
```

<Info>
  Unpublishing doesn't delete the tour - it simply hides it from public view. Admins can still see unpublished tours.
</Info>

Some contexts show newest tours:

```php
Tours::latest()->limit(3)->get()
```

**Used for:**
- Booking page suggestions
- New offerings highlight
- Admin management views
