## Overview

Safaris Web features rich content through travel blogs and video media to inspire and inform your Kenya safari planning. Learn about destinations, wildlife, culture, and travel tips through engaging articles and videos.

## Blog Posts

### Blog Structure

Blogs are managed through the `Blog` model (`app/Models/Blog.php`) and stored in the `blogs` database table.

**Database Schema:**

<Tabs>
  <Tab title="Core Fields">
    | Field | Type | Required | Description |
    |-------|------|----------|-------------|
    | `id` | Integer | Auto | Unique identifier |
    | `title` | Text | Yes | Blog post title |
    | `content` | Text | Yes | Full article content |
    | `image` | Text | Yes | Featured image |
  </Tab>
  <Tab title="Additional Images">
    Blog posts support multiple images (similar to tours):
    - `image` - Required featured image
    - `image2` - Required secondary image
    - `image3` - Optional image
    - `image4` - Optional image
    - `image5` - Optional image
    - `image6` - Optional image
    
    <Tip>
      Up to 6 images per blog post for comprehensive visual storytelling
    </Tip>
  </Tab>
  <Tab title="Management">
    | Field | Type | Default |
    |-------|------|--------|
    | `user_id` | Integer | - |
    | `published` | Integer | 0 |
    | `created_at` | Timestamp | Auto |
    | `updated_at` | Timestamp | Auto |
  </Tab>
</Tabs>

### Blog Model

From `app/Models/Blog.php`:

```php
protected $fillable = [ 
    'image',
    'title',
    'content'
];
```

**Fillable Fields:**
- Core content fields for mass assignment
- Additional images and management fields controlled separately

## Browsing Blog Posts

### Main Blog Page

**Route:** `/all-blogs`

**Controller:** `BlogController::blogs()`

**Display Logic:**
```php
$blogs = Blog::latest()->paginate(10);
```

**Features:**
- 10 blog posts per page
- Latest posts first
- Pagination controls
- Featured images displayed
- Post titles and excerpts

<Info>
  The route is `/all-blogs` (not just `/blogs`) to distinguish from the admin management route.
</Info>

### Blog Index Page

**Route:** `/blogs`

**Controller:** `BlogController::index()`

**Display:**
```php
$coverBlogs = Blog::latest()->paginate(6);
$blogs = Blog::latest()->limit(2)->get();
```

**Shows:**
- 6 cover blogs (paginated)
- 2 latest blogs highlighted
- Multiple display formats

### Homepage Blog Integration

Blogs appear throughout the platform:

**Homepage:**
```php
$blogs = Blog::latest()->limit(2)->get();
$coverBlogs = Blog::latest()->limit(4)->get();
```

**Safaris Page:**
```php
$blogs = Blog::latest()->limit(2)->get();
```

**Search Results:**
```php
$blogs = Blog::latest()->limit(2)->get();
```

<Note>
  Latest 2 blog posts appear on most major pages for continuous inspiration and information.
</Note>

## Admin Blog Management

### Blog Administration Routes

<Tabs>
  <Tab title="View All">
    **Admin Management:**
    - Route: `/blogs` (admin view)
    - Controller: `BlogController::blogs()`
    - Display: `Blog::latest()->paginate(10)`
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Create">
    **New Blog Post:**
    - Form: `/blogs/create` (GET)
    - Submit: `/blogs/store` (POST)
    - Middleware: `auth`, `verified`
    - Validation:
      ```php
      'title' => 'required',
      'image' => 'required',
      'content' => 'required'
      ```
  </Tab>
  <Tab title="Edit">
    **Update Blog:**
    - Form: `/blogs/edit/{id}` (GET)
    - Update: `/blogs/update/{id}` (POST)
    - Middleware: `auth`, `verified`
    - Can update title, content, images
  </Tab>
  <Tab title="Publish">
    **Toggle Status:**
    - Route: `/blogs/publish/{id}` (PUT)
    - Toggles `published` between 0 and 1
    - Middleware: `auth`, `verified`
  </Tab>
  <Tab title="Delete">
    **Remove Blog:**
    - Route: `/blogs/delete/{id}` (DELETE)
    - Permanently removes blog post
    - Middleware: `auth`, `verified`
    - Success: "Content Deleted"
  </Tab>
</Tabs>

### Blog Creation Workflow

<Steps>
  <Step title="Create Post">
    Admin writes blog with:
    - Engaging title
    - Comprehensive content
    - Featured image (required)
    - Secondary image (required)
    - Up to 4 additional images
  </Step>
  <Step title="Draft Status">
    Post created with `published = 0`
  </Step>
  <Step title="Review">
    Check for:
    - Content accuracy
    - Grammar and spelling
    - Image quality
    - Information relevance
  </Step>
  <Step title="Publish">
    Set `published = 1` to make visible
  </Step>
  <Step title="Distribution">
    Published blog appears:
    - On `/all-blogs` page
    - In homepage blog section
    - On safari/search pages (latest 2)
  </Step>
</Steps>

<Warning>
  Blog images require 2 minimum (image and image2). Controller attempts to store image2, so it should be provided.
</Warning>

## Video Content

### Video Structure

Videos are managed through the `Video` model (`app/Models/Video.php`) and stored in the `videos` database table.

**Database Schema:**

<Tabs>
  <Tab title="Video Data">
    | Field | Type | Required | Description |
    |-------|------|----------|-------------|
    | `id` | Integer | Auto | Unique identifier |
    | `url` | String | Yes | Video URL (YouTube, Vimeo, etc.) |
    | `created_at` | Timestamp | Auto | Upload date |
    | `updated_at` | Timestamp | Auto | Last update |
  </Tab>
</Tabs>

**Video Model:**

From `app/Models/Video.php`:

```php
protected $fillable = [
    'url'
];
```

<Info>
  Videos use external hosting (YouTube, Vimeo) with URLs stored in the database. This keeps the platform lightweight while providing rich video content.
</Info>

### Browsing Videos

**Route:** `/videos`

**Controller:** `VideoController::index()`

**Features:**
- All published videos displayed
- Video embeds from external platforms
- Grid or list layout
- Latest videos first

### Homepage Video Display

**Homepage Route:** `/`

**Query:**
```php
$videos = Video::inRandomOrder()->limit(3)->get();
```

**Features:**
- 3 random videos on homepage
- Variety on each visit
- Embedded video players
- Quick visual engagement

<Tip>
  Random video selection ensures different content is highlighted on each homepage visit.
</Tip>

## Content Publishing Workflow

### For Blog Posts

<Steps>
  <Step title="Content Creation">
    Admin writes comprehensive article with:
    - Research and accurate information
    - Engaging writing style
    - Helpful tips and advice
    - Personal insights
  </Step>
  <Step title="Image Selection">
    Choose high-quality images:
    - 2 minimum required
    - Up to 6 total
    - Relevant to content
    - High resolution
  </Step>
  <Step title="Draft Review">
    Check content quality:
    - Accuracy verification
    - Grammar and spelling
    - Image quality
    - Link functionality
  </Step>
  <Step title="SEO Optimization">
    - Descriptive title
    - Keyword-rich content
    - Alt text for images (if implemented)
  </Step>
  <Step title="Publish">
    Toggle `published` to 1, making blog live across platform
  </Step>
</Steps>

### For Videos

<Steps>
  <Step title="Video Production">
    Create or source video content:
    - Professional quality
    - Relevant to Kenya tourism
    - Appropriate length (2-10 minutes typically)
  </Step>
  <Step title="External Hosting">
    Upload to YouTube or Vimeo:
    - Public or unlisted
    - Embeddable enabled
    - Good metadata (title, description, tags)
  </Step>
  <Step title="URL Extraction">
    Get embeddable video URL
  </Step>
  <Step title="Database Entry">
    Add video to platform via `/videos/create`:
    - Paste URL
    - Submit
  </Step>
  <Step title="Verification">
    Check video displays correctly on frontend
  </Step>
</Steps>
