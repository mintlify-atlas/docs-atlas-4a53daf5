## Overview

The Tours Management system allows you to create and manage tour packages with pricing, images, categories, and detailed information. All tour management operations are handled by the `ToursController`.

**Controller:** `App\Http\Controllers\ToursController`  
**Model:** `App\Models\Tours`  
**Route Prefix:** `/addtour`

<Note>
  All tour management operations require `['auth', 'verified']` middleware - you must be logged in and have a verified email.
</Note>

## Tour Routes

| Action | Route | Name | Method | Middleware |
|--------|-------|------|--------|------------|
| List all tours | `/addtour` | `tours` | GET | `['auth', 'verified']` |
| View tour details | `/addtour/show/{id}` | `addtour.show` | GET | None |
| Create tour form | `/addtour/create` | `addtour.create` | GET | `['auth', 'verified']` |
| Store new tour | `/addtour/store` | `addtour.store` | POST | `['auth', 'verified']` |
| Edit tour form | `/addtour/edit/{id}` | `addtour.edit` | GET | `['auth', 'verified']` |
| Update tour | `/addtour/update/{id}` | `addtour.update` | POST | `['auth', 'verified']` |
| Delete tour | `/addtour/delete/{id}` | `addtour.delete` | DELETE | `['auth', 'verified']` |
| Publish/unpublish | `/addtour/publish/{id}` | `addtour.publish` | PUT | `['auth', 'verified']` |

### Store New Tour

**Route:** `/addtour/store`  
**Method:** POST  
**Controller Method:** `store(Request $request)`  
*Source: ToursController.php:63-135*

#### Validation Rules

```php
$request->validate([
    'hotel' => 'required',
    'transport' => 'required',
    'image' => 'required|mimes:jpg,jpeg,svg,png,webp',
    'image1' => 'required|mimes:jpg,jpeg,svg,png,webp',
    'single_room' => 'required',
    'per_person_sharing' => 'required',
    'meals' => 'required',
    'category' => 'required',
    'sub_category' => 'required',
    'location' => 'required',
]);
```

**Accepted Image Formats:** jpg, jpeg, svg, png, webp

#### Image Handling

The system handles up to 7 images:

```php
// Required images
$path = $request->image->store('images', 'public');
$path1 = $request->image1->store('images', 'public');
$tours->image = $path;
$tours->image1 = $path1;

// Optional images (image2 through image6)
if($request->image2) {
    $path2 = $request->image2->store('images', 'public');
    $tours->image2 = $path2;
}
```

**Image Storage:**
- Stored in `storage/app/public/images/`
- Accessible via `storage/images/` symlink
- Automatic filename generation

#### User Association

The authenticated user's ID is automatically attached:

```php
$tours->user_id = Auth::id();
```
*Source: ToursController.php:130*

#### Success Response

After successful creation:

```php
return redirect()->route('tours')
    ->with('success', 'You Added a Tour.');
```

#### Image Update Logic

```php
if($request->hasFile('image') || $request->hasFile('image2') || 
   $request->hasFile('image3') || $request->hasFile('image4') || 
   $request->hasFile('image5') || $request->hasFile('image6')) 
{
    // Store new images
    $path = $request->image->store('images', 'public');
    $tours->image = $path;
    // ... etc
} else {
    // Keep existing images
    unset($tours['image']);
    unset($tours['image2']);
    // ... etc
}
```

## Publishing Workflow

### Publish/Unpublish Tour

**Route:** `/addtour/publish/{id}`  
**Method:** PUT  
**Controller Method:** `publish($id)`  
*Source: ToursController.php:260-273*

```php
public function publish($id)
{
    $tour = Tours::find($id);
    
    if ($tour->published == 0) {
        $tour->published = 1;  // Publish
        $tour->update();
    } else {
        $tour->published = 0;  // Unpublish
        $tour->update();
    }
    
    return redirect()->route('tours');
}
```

**Publication States:**
- `published = 0` - Draft/Unpublished (not visible to public)
- `published = 1` - Published (visible to public)

<Warning>
  The publish endpoint toggles the published state. Make sure to check the current state before calling this endpoint.
</Warning>

### How to Use

<Tabs>
  <Tab title="Publish a Draft">
    ```bash
    # PUT request to publish tour with ID 5
    PUT /addtour/publish/5
    ```
    
    If `published = 0`, it will be set to `1`
  </Tab>
  
  <Tab title="Unpublish a Tour">
    ```bash
    # PUT request to unpublish tour with ID 5
    PUT /addtour/publish/5
    ```
    
    If `published = 1`, it will be set to `0`
  </Tab>
</Tabs>

## Deleting a Tour

**Route:** `/addtour/delete/{id}`  
**Method:** DELETE  
**Controller Method:** `destroy($id)`  
*Source: ToursController.php:249-257*

```php
public function destroy(Tours $tours, $id)
{
    $tours = Tours::find($id);
    $tours->delete();
    
    return redirect()->route('tours')
        ->with('success', 'You Deleted a Tour.');
}
```

<Warning>
  **Deletion is permanent!** There is no soft delete implemented. Once a tour is deleted, it cannot be recovered.
</Warning>

<Steps>
  <Step title="Locate the Tour">
    Find the tour you want to delete in the tours list at `/addtour`
  </Step>
  
  <Step title="Confirm Deletion">
    Click the delete button (should show confirmation dialog)
  </Step>
  
  <Step title="Tour Removed">
    The tour is permanently deleted from the database and you're redirected to the tours list
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Image Optimization" icon="image">
    - Compress images before uploading
    - Use web-optimized formats (webp, jpg)
    - Recommended dimensions: 1200x800px for main images
    - Keep file sizes under 500KB
    - At minimum, provide 2 images (required)
  </Accordion>
  
  <Accordion title="Pricing Information" icon="dollar-sign">
    - Clearly specify what's included in the price
    - Use consistent currency formatting
    - Explain difference between single room and per person sharing
    - Update prices regularly
  </Accordion>
  
  <Accordion title="Category Selection" icon="folder">
    - Choose the most relevant category
    - Use subcategories to further organize tours
    - Create new categories if needed before adding tours
    - Be consistent with category naming
  </Accordion>
  
  <Accordion title="Descriptions and Information" icon="text">
    - Use the `additional_info` field for detailed tour descriptions
    - Include itinerary details
    - Mention included activities
    - Specify tour duration
    - Add booking policies
  </Accordion>
  
  <Accordion title="Publishing Strategy" icon="rocket">
    - Create tours as drafts first (`published = 0`)
    - Review all information before publishing
    - Test tour display on public pages
    - Use publish toggle to temporarily hide tours
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Image Upload Fails" icon="exclamation-triangle">
    **Possible causes:**
    - File format not supported (must be jpg, jpeg, svg, png, or webp)
    - File size too large (check PHP upload limits)
    - Storage directory not writable
    
    **Solutions:**
    - Verify image format
    - Check `storage/app/public` permissions
    - Run `php artisan storage:link` to create symlink
    - Check `php.ini` settings: `upload_max_filesize` and `post_max_size`
  </Accordion>
  
  <Accordion title="Validation Errors" icon="times-circle">
    **Required fields:**
    - Hotel, transport, meals
    - Single room price, per person sharing price
    - Category, subcategory, location
    - Image and image1 (when creating)
    
    **Note:** All fields must be filled. Empty strings won't pass validation.
  </Accordion>
  
  <Accordion title="Tour Not Visible on Public Site" icon="eye-slash">
    **Check:**
    - Is `published = 1`? Tours with `published = 0` are drafts
    - Is the category active and properly configured?
    - Clear application cache: `php artisan cache:clear`
  </Accordion>
  
  <Accordion title="Cannot Edit or Delete Tour" icon="lock">
    **Requirements:**
    - Must be logged in (`auth` middleware)
    - Email must be verified (`verified` middleware)
    - Tour must exist in database
    
    If you see "403 Forbidden" or are redirected to login, verify your authentication status.
  </Accordion>
</AccordionGroup>
