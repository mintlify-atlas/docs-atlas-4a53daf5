## Introduction

The Safaris Web admin dashboard provides a comprehensive content management system for managing tours, bookings, blog posts, videos, categories, and customer enquiries. All administrative functions require authentication and email verification.

## Accessing the Admin Dashboard

The admin dashboard can be accessed via two routes:

- `/admin` - Main admin page (route: `admin`)
- `/dashboard` - Alternative dashboard route (route: `dashboard`)

Both routes are protected by the `auth` middleware, requiring users to be logged in.

**Route Definition:**
```php
Route::get('/admin', [ToursController::class, 'admin'])
  ->name('admin')
  ->middleware('auth');

Route::get('/dashboard', [HomeController::class, 'admin'])
  ->name('dashboard')
  ->middleware('auth');
```

## Navigation Structure

The admin interface is organized into the following sections:

1. **Dashboard** - Overview of system activity
2. **Tours** - Tour package management (`/addtour`)
3. **Bookings** - Booking management (`/bookings`)
4. **Blog** - Blog post management (`/blogs`, `/all-blogs`)
5. **Videos** - Video content management (`/videos`)
6. **Categories** - Category management (`/categories`)
7. **Enquiries** - Customer enquiry management (`/enquiries`)
8. **Users** - User account management (`/users`)
9. **Profile** - Admin profile and personal bookings (`/profile`)

## Middleware Protection

Most admin routes are protected by two middleware layers:

<AccordionGroup>
  <Accordion title="auth - Authentication Middleware">
    Ensures users are logged in before accessing admin functionality.
    
    **Redirects to:** Login page if not authenticated
  </Accordion>
  
  <Accordion title="verified - Email Verification Middleware">
    Requires users to verify their email address before performing critical operations.
    
    **Protected operations:** Create, update, delete operations for all content types
    
    **Redirects to:** Email verification notice if not verified
  </Accordion>
</AccordionGroup>
