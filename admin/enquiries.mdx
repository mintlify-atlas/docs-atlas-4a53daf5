## Overview

The Enquiries Management system allows you to receive, view, and manage customer inquiries about tours and travel packages. Enquiries are submitted through a public form and stored for admin review. All enquiry operations are handled by the `EnquiryController`.

**Controller:** `App\Http\Controllers\EnquiryController`  
**Model:** `App\Models\Enquiry`  
**Table:** `enquiries`  
**Route:** `/enquiries` (resource route)

<Warning>
  Enquiry routes use Laravel's resource controller pattern and are **NOT protected** by authentication middleware. Anyone can view, create, and potentially delete enquiries.
</Warning>

## Enquiry Routes

Enquiries use Laravel's resource routing:

```php
Route::resource('enquiries', EnquiryController::class);
```
*Source: web.php:53*

This generates the following routes:

| Action | Route | Name | Method | Middleware |
|--------|-------|------|--------|------------|
| List all enquiries | `/enquiries` | `enquiries.index` | GET | **None** |
| Create enquiry form | `/enquiries/create` | `enquiries.create` | GET | **None** |
| Store new enquiry | `/enquiries` | `enquiries.store` | POST | **None** |
| View single enquiry | `/enquiries/{id}` | `enquiries.show` | GET | **None** |
| Edit enquiry form | `/enquiries/{id}/edit` | `enquiries.edit` | GET | **None** |
| Update enquiry | `/enquiries/{id}` | `enquiries.update` | PUT/PATCH | **None** |
| Delete enquiry | `/enquiries/{id}` | `enquiries.destroy` | DELETE | **None** |

<Note>
  The create and store routes are intentionally public (for customer submissions), but the other routes should be protected.
</Note>

## Deleting an Enquiry

**Route:** `/enquiries/{id}`  
**Method:** DELETE  
**Controller Method:** `destroy(Enquiry $enquiry)`  
*Source: EnquiryController.php:133-141*

```php
public function destroy(Enquiry $enquiry)
{
    $enquiry = Enquiry::find($enquiry->id);
    $enquiry->delete();
    
    return redirect()->route('enquiries.index')
        ->with('success', 'Enquiry Deleted Successfully');
}
```

<Warning>
  **Issues:**
  1. No authentication required - anyone can delete enquiries
  2. Hard delete - enquiries are permanently removed
  3. No authorization check
</Warning>

## Enquiry Workflow

<Steps>
  <Step title="Customer Submits Enquiry">
    Customer fills out enquiry form with travel details and preferences
    - No account required

  <Step title="Admin Reviews Enquiry">

    Once resolved:
    - Delete spam enquiries
    - Archive completed enquiries (if soft delete implemented)
    - Keep for future reference or analysis
  </Step>
</Steps>

## Security Issues

<Warning>
  **CRITICAL SECURITY AND PRIVACY ISSUES**
  
  The enquiries system has severe vulnerabilities:
</Warning>

### No Authentication on Admin Routes

**Problem:** Anyone can view all enquiries with personal information

```php
// Current (INSECURE)
Route::resource('enquiries', EnquiryController::class);
```

**Impact:**
- Public access to customer names, emails, phone numbers
- Privacy violation (GDPR/data protection concerns)
- Potential for data scraping
- Competitive intelligence risk

**Required Fix:**

```php
// Public routes (for customer submissions)
Route::get('/enquiries/create', [EnquiryController::class, 'create'])
    ->name('enquiries.create');
Route::post('/enquiries', [EnquiryController::class, 'store'])
    ->name('enquiries.store');

// Protected routes (for admin only)
Route::middleware(['auth', 'verified'])->group(function () {
    Route::get('/enquiries', [EnquiryController::class, 'index'])
        ->name('enquiries.index');
    Route::get('/enquiries/{id}', [EnquiryController::class, 'show'])
        ->name('enquiries.show');
    Route::delete('/enquiries/{id}', [EnquiryController::class, 'destroy'])
        ->name('enquiries.destroy');
});
```

### No Spam Protection

**Problem:** Enquiry form has no CAPTCHA or rate limiting

**Impact:**
- Vulnerable to spam submissions
- Can be abused by bots
- Database pollution

**Recommended Fix:**

```php
// Add to validation
$request->validate([
    // ... other fields
    'g-recaptcha-response' => 'required|captcha',
]);

// Add to route
Route::post('/enquiries', [EnquiryController::class, 'store'])
    ->middleware('throttle:5,1'); // 5 submissions per minute
```

### Missing Functionality

**Problem:** Edit and update methods are empty

**Impact:**
- Cannot update enquiry status
- Cannot add admin notes
- Cannot track follow-up actions

**Recommended Implementation:**

Add status tracking:

// Migration
$table->enum('status', ['new', 'in_progress', 'contacted', 'converted', 'closed'])
      ->default('new');
$table->text('admin_notes')->nullable();

## Integration with Bookings

<Note>
  Currently, enquiries and bookings are separate systems. Consider implementing a workflow to convert enquiries into bookings.
</Note>

### Recommended Conversion Flow

```php
// Add method to convert enquiry to booking
public function convertToBooking(Enquiry $enquiry)
{
    $this->authorize('convert', $enquiry);
    
    // Create booking from enquiry
    $booking = new Booking([
        'first_name' => explode(' ', $enquiry->full_name)[0],
        'last_name' => explode(' ', $enquiry->full_name, 2)[1] ?? '',
        'email' => $enquiry->email_address,
        'phone_number' => $enquiry->phone_number,
        'arrival_date' => $enquiry->travel_date,
        'adults' => $enquiry->adults,
        'kids' => $enquiry->kids,
        'country' => $enquiry->nationality,
        'destination' => $enquiry->destinations,
    ]);
    
    $booking->save();
    
    // Update enquiry status
    $enquiry->status = 'converted';
    $enquiry->save();
    
    return redirect()->route('bookings.edit', $booking->id)
        ->with('success', 'Enquiry converted to booking');
}
```

  <Accordion title="Enquiries Not Appearing" icon="exclamation-triangle">

    - Test database connection

       ```
       MAIL_MAILER=smtp
       MAIL_HOST=smtp.mailtrap.io
       MAIL_PORT=2525
       MAIL_USERNAME=your-username
       MAIL_PASSWORD=your-password
       ```

       ```bash
       php artisan tinker
       Mail::raw('Test', function($msg) { $msg->to('test@example.com'); });
       ```
  </Accordion>
