## Overview

Safaris Web uses Laravel's built-in authentication system with email verification enabled. All admin operations require users to be authenticated and have verified their email addresses.

## Authentication Routes

The application uses Laravel's default authentication routes with email verification:

```php
// Enable authentication routes with verification
Auth::routes(['verify' => true]);
```

This generates the following routes:

### Login Routes

| Method | Route | Name | Description |
|--------|-------|------|-------------|
| GET | `/login` | `login` | Display login form |
| POST | `/login` | - | Process login credentials |
| POST | `/logout` | `logout` | Log out current user |

### Registration Routes

| Method | Route | Name | Description |
|--------|-------|------|-------------|
| GET | `/register` | `register` | Display registration form |
| POST | `/register` | - | Process registration |

### Password Reset Routes

| Method | Route | Name | Description |
|--------|-------|------|-------------|
| GET | `/password/reset` | `password.request` | Display reset form |
| POST | `/password/email` | `password.email` | Send reset link |
| GET | `/password/reset/{token}` | `password.reset` | Reset with token |
| POST | `/password/reset` | `password.update` | Update password |

## Email Verification

<Note>
  Email verification is **required** for all create, update, and delete operations in the admin dashboard.
</Note>

### Email Verification Routes

The application implements custom email verification routes:

#### Verification Notice

```php
Route::get('/email/verify', function () {
    return view('auth.verify');
})->middleware('auth')->name('verification.notice');
```

**Route:** `/email/verify`  
**Name:** `verification.notice`  
**Middleware:** `auth`  
**Purpose:** Display email verification notice to users who haven't verified their email

#### Verification Link Handler

```php
Route::get('/email/verify/{id}/{hash}', function (EmailVerificationRequest $request) {
    $request->fulfill();
    return redirect('/');
})->middleware(['auth', 'signed'])->name('verification.verify');
```

**Route:** `/email/verify/{id}/{hash}`  
**Name:** `verification.verify`  
**Middleware:** `auth`, `signed`  
**Purpose:** Process email verification when user clicks link in email

<Steps>
  <Step title="User clicks verification link in email">
    The link contains a signed URL with user ID and hash
  </Step>
  
  <Step title="Request is verified">
    Laravel validates the signature and marks email as verified
  </Step>
  
  <Step title="User is redirected">
    Successful verification redirects to homepage `/`
  </Step>
</Steps>

#### Resend Verification Email

```php
Route::post('/email/verification-notification', function (Request $request) {
    $request->user()->sendEmailVerificationNotification();
    return back()->with('message', 'Verification link sent!');
})->middleware(['auth', 'throttle:6,1'])->name('verification.send');
```

**Route:** `/email/verification-notification`  
**Name:** `verification.send`  
**Method:** POST  
**Middleware:** `auth`, `throttle:6,1`  
**Rate Limit:** 6 requests per minute  
**Purpose:** Resend verification email to user

## User Model Configuration

The User model implements the `MustVerifyEmail` contract:

```php
use Illuminate\Contracts\Auth\MustVerifyEmail;

class User extends Authenticatable implements MustVerifyEmail
{
    protected $fillable = [
        'name',
        'email',
        'password',
    ];
}
```

### User Attributes

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | User's full name |
| `email` | string | User's email address (must be unique) |
| `password` | string | Hashed password |
| `email_verified_at` | datetime | Timestamp of email verification |
| `remember_token` | string | Token for "remember me" functionality |

## Authentication Workflow

<Steps>
  <Step title="Register an Account">
    Navigate to `/register` and create an account with:
    - Name
    - Email address
    - Password
    - Password confirmation
  </Step>
  
  <Step title="Verify Your Email">
    Check your email for a verification link and click it. The link will direct you to `/email/verify/{id}/{hash}`.
  </Step>
  
  <Step title="Access Admin Dashboard">
    Once verified, navigate to `/admin` or `/dashboard` to access administrative features.
  </Step>
  
  <Step title="Manage Content">
    You can now create, edit, and delete tours, blogs, videos, categories, and manage bookings.
  </Step>
</Steps>

<Warning>
  If you attempt to access protected routes without email verification, you will be redirected to `/email/verify` with a notice to verify your email.
</Warning>

## Rate Limiting

The verification email resend endpoint is rate-limited to prevent abuse:

- **Limit:** 6 requests per minute per user
- **Middleware:** `throttle:6,1`

If you exceed this limit, you'll receive a "Too Many Requests" error and must wait before requesting another verification email.
