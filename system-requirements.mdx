## Account Requirements

### Registration Prerequisites

<Steps>
  <Step title="Valid Email Address">
    You need a working email address that can receive verification emails. The platform uses Laravel's `MustVerifyEmail` interface.
    
    **Email is used for:**
    - Account verification (route: `/email/verify/{id}/{hash}`)
    - Booking confirmations
    - Password reset functionality
    - Communication from Safaris Web team
  </Step>
  <Step title="Password Requirements">
    Create a secure password meeting these criteria:
    - Minimum 8 characters
    - Stored securely using Laravel's password hashing
    - Password confirmation required during registration
  </Step>
  <Step title="Personal Information">
    Required for registration:
    - Full name
    - Email address
    - Password
    
    Additional information needed for bookings:
    - Phone number
    - Country of residence
  </Step>
</Steps>

### Email Verification

<Warning>
  **Verification is mandatory** for booking functionality. Routes with `verified` middleware cannot be accessed without email verification.
</Warning>

Protected routes requiring verification:
- `/bookings/new` - Submit new booking (POST)
- `/profile` - View your profile and bookings
- `/bookings/edit/{id}` - Edit existing booking
- All admin routes (if applicable)

**Verification Process:**

1. Register at `/register`
2. Check email for verification link
3. Click the link (format: `/email/verify/{id}/{hash}`)
4. Automatic redirect to home page upon success
5. Can now access all booking features

<Tip>
  Didn't receive the verification email? Request a new one at `/email/verification-notification` (throttled to 6 requests per minute).
</Tip>

## Network Requirements

<Info>
  Recommended minimum connection speed: 2 Mbps for smooth browsing and image loading.
</Info>

**Data Usage Estimates:**
- Homepage: ~2-3 MB (includes images from tours, blogs, videos)
- Safaris listing page: ~1-2 MB per page (20 tours per page)
- Individual tour page: ~500 KB - 1 MB (depends on number of images)
- Booking process: ~200-300 KB

**Why bandwidth matters:**
- Tours include multiple high-quality images (up to 6 per tour)
- Video content available at `/videos` route
- Blog posts may include images
- Homepage displays highlights, products, services, accommodations, and events

## Security Requirements

### HTTPS Connection

<Check>The platform should be accessed via HTTPS for secure data transmission</Check>

**Protected by HTTPS:**
- Login credentials
- Personal information during registration
- Booking details and transaction codes
- Password reset tokens

### CSRF Protection

<Info>
  Laravel's CSRF protection is enabled. Forms automatically include CSRF tokens for security.
</Info>

### reCAPTCHA

Booking forms include Google reCAPTCHA:
- Validation rule: `'g-recaptcha-response' => 'required|captcha'`
- Prevents automated booking submissions
- Protects against spam and abuse

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Can't Login After Registration">
    **Issue**: Account created but login fails
    
    **Solution**: Check if you verified your email. Visit `/email/verify` to check verification status or request a new link.
  </Accordion>
  <Accordion title="Booking Form Won't Submit">
    **Possible causes:**
    - Email addresses don't match (validation checks `email1 == email2`)
    - reCAPTCHA not completed
    - Missing required fields
    - Session expired (login again)
    - Not verified (check email verification)
  </Accordion>
  <Accordion title="Images Not Loading">
    **Possible causes:**
    - Slow internet connection
    - Browser cache issues (try clearing cache)
    - Images stored in `storage/images` - ensure public storage linked
  </Accordion>
  <Accordion title="Search Not Working">
    **Solution**: Ensure JavaScript is enabled. Search functionality at `/search` route uses Laravel query builder with LIKE operators.
  </Accordion>
</AccordionGroup>
