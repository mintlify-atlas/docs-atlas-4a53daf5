## Environment File

Safaris Web uses Laravel's `.env` file for environment-specific configuration. All sensitive credentials and settings should be stored here.

<Warning>
Never commit your `.env` file to version control. It contains sensitive credentials.
</Warning>

## Core Application Settings

### Application Configuration

```bash .env
APP_NAME="Safaris Web"
APP_ENV=local
APP_KEY=base64:your-generated-key-here
APP_DEBUG=true
APP_URL=http://localhost:8000
```

**Key Settings:**
- `APP_ENV`: Environment (local, staging, production)
- `APP_DEBUG`: Enable debug mode (set to `false` in production)
- `APP_KEY`: Generated via `php artisan key:generate`
- `APP_URL`: Base URL for your application

## Database Configuration

Safaris Web supports MySQL/MariaDB and PostgreSQL databases.

<CodeGroup>

```bash MySQL/MariaDB
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=safaris_web
DB_USERNAME=root
DB_PASSWORD=your_password
```

```bash PostgreSQL
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=safaris_web
DB_USERNAME=postgres
DB_PASSWORD=your_password
```

</CodeGroup>

### Database Setup Steps

1. **Create the database:**
   ```bash
   mysql -u root -p
   CREATE DATABASE safaris_web CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

2. **Run migrations:**
   ```bash
   php artisan migrate
   ```

3. **Verify connection:**
   ```bash
   php artisan tinker
   DB::connection()->getPdo();
   ```

## Mail Configuration

The application uses Laravel's mail system for sending booking confirmations and notifications.

### SMTP Configuration

<CodeGroup>

```bash Gmail
MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-app-password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=your-email@gmail.com
MAIL_FROM_NAME="${APP_NAME}"
```

```bash Mailgun
MAIL_MAILER=mailgun
MAILGUN_DOMAIN=your-domain.mailgun.org
MAILGUN_SECRET=your-mailgun-secret
MAIL_FROM_ADDRESS=no-reply@yourdomain.com
MAIL_FROM_NAME="${APP_NAME}"
```

```bash Mailtrap (Testing)
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=your-mailtrap-username
MAIL_PASSWORD=your-mailtrap-password
MAIL_ENCRYPTION=tls
```

</CodeGroup>

<Note>
For Gmail, you must generate an App Password from your Google Account settings if 2FA is enabled.
</Note>

## Google reCAPTCHA Setup

Safaris Web uses Google reCAPTCHA v2 to protect booking and enquiry forms from spam.

### Configuration

Add these variables to your `.env` file:

```bash .env
NOCAPTCHA_SITEKEY=your-recaptcha-site-key
NOCAPTCHA_SECRET=your-recaptcha-secret-key
```

### Getting reCAPTCHA Keys

1. Visit [Google reCAPTCHA Admin Console](https://www.google.com/recaptcha/admin)
2. Register a new site:
   - **Label**: Safaris Web
   - **reCAPTCHA type**: reCAPTCHA v2 ("I'm not a robot" Checkbox)
   - **Domains**: Add your domain(s)
3. Copy the Site Key and Secret Key to your `.env` file

### Usage in Code

The reCAPTCHA is implemented in booking forms (`~/workspace/source/app/Http/Controllers/BookingController.php:76`):

```php
$request->validate([
    'first_name' => 'required',
    'last_name' => 'required',
    // ... other fields
    'g-recaptcha-response' => 'required|captcha',
]);
```

## File Storage Configuration

Safaris Web stores uploaded images in the `storage/app/public` directory.

```bash .env
FILESYSTEM_DISK=public
```

**Create the storage link:**
```bash
php artisan storage:link
```

This creates a symbolic link from `public/storage` to `storage/app/public`.

### Image Upload Locations

Based on the codebase, images are stored for:
- **Tours**: `storage/app/public/images/` (ToursController.php:80)
- **Blogs**: `storage/app/public/images/`
- **Products**: `storage/app/public/images/`
- **Events**: `storage/app/public/images/`
- **Accommodations**: `storage/app/public/images/`

## Logging Configuration

```bash .env
LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug
```

**Available channels:**
- `stack`: Multiple channels
- `single`: Single log file
- `daily`: Daily rotating log files
- `slack`: Send logs to Slack

## Production Configuration

When deploying to production, update these settings:

```bash .env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com

# Enable optimizations
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

<Warning>
**Security Checklist for Production:**
- Set `APP_DEBUG=false`
- Use strong `DB_PASSWORD`
- Enable HTTPS (APP_URL)
- Set secure session cookies
- Configure proper file permissions (755 for directories, 644 for files)
- Keep `composer.lock` in version control
</Warning>
