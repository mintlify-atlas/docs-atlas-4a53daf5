## Overview

Safaris Web uses Laravel migrations to manage database schema. All migration files are located in `~/workspace/source/database/migrations/`.

## Running Migrations

### Execute All Migrations

```bash
php artisan migrate
```

### Rollback Migrations

<CodeGroup>

```bash Rollback Last Batch
php artisan migrate:rollback
```

```bash Rollback All
php artisan migrate:reset
```

```bash Fresh Migration
php artisan migrate:fresh
```

```bash Fresh with Seeding
php artisan migrate:fresh --seed
```

</CodeGroup>

## Database Schema

### Users Table

**Migration**: `2014_10_12_000000_create_users_table.php`

```php
Schema::create('users', function (Blueprint $table) {
    $table->id();
    $table->string('name');
    $table->string('email')->unique();
    $table->timestamp('email_verified_at')->nullable();
    $table->string('password');
    $table->boolean('isAdmin')->default(false);
    $table->rememberToken();
    $table->timestamps();
});
```

**Purpose**: Stores user accounts with email verification support and admin flag.


### Creating a Seeder

```bash
php artisan make:seeder CategorySeeder
```

Example seeder implementation:

```php
use App\Models\Category;
use Illuminate\Database\Seeder;

class CategorySeeder extends Seeder
{
    public function run()
    {
        Category::create([
            'name' => 'Safari Tours',
            'sub_name' => 'Wildlife Safari',
            'image' => 'categories/safari.jpg',
            'desc' => 'Experience African wildlife'
        ]);
    }
}
```

## Schema Modification Patterns

### Adding Columns

Notice how image fields were added to existing tables:

```php
// Migration: 2023_02_19_082113_add_image_fields_to_tours_table.php
Schema::table('tours', function (Blueprint $table) {
    $table->text('image3')->nullable();
    $table->text('image4')->nullable();
    $table->text('image5')->nullable();
    $table->text('image6')->nullable();
});
```

### Adding Publish Status

```php
// Migration: 2023_01_14_215019_add_publish_to_tours.php
Schema::table('tours', function (Blueprint $table) {
    $table->integer('published')->default(0);
    $table->integer('user_id');
});
```

## Database Relationships

The schema supports these relationships:

- **User → Tour Bookings**: One-to-Many (via `user_id`)
- **User → Tours**: One-to-Many (creator relationship)
- **User → Blogs**: One-to-Many (author relationship)
- **Categories → Tours**: Referenced via `category` field

See [Models & Relationships](/development/models-relationships) for implementation details.

## Indexing & Performance

### Recommended Indexes

```php
// Add indexes for frequently queried columns
$table->index('category');
$table->index('published');
$table->index('user_id');
$table->index(['category', 'published']);
```

### Full-Text Search

For tour search functionality:

```php
$table->fullText(['hotel', 'location', 'additional_info']);
```

## Migration Best Practices

1. **Never modify existing migrations** - Create new migrations for changes
2. **Use descriptive names** - `add_image_fields_to_tours_table.php`
3. **Make migrations reversible** - Always implement `down()` method
4. **Test rollbacks** - Ensure `migrate:rollback` works correctly
5. **Use transactions** - Wrap schema changes in transactions when possible
