## User Model

**Location**: `~/workspace/source/app/Models/User.php`

```php
class User extends Authenticatable implements MustVerifyEmail
{
    use HasApiTokens, HasFactory, Notifiable;

    protected $fillable = [
        'name',
        'email',
        'password',
    ];

    protected $hidden = [
        'password',
        'remember_token',
    ];

    protected $casts = [
        'email_verified_at' => 'datetime',
    ];

    // Relationships
    public function bookings()
    {
        return $this->hasMany('App\Booking');
    }
}
```

### Key Features

- **Implements `MustVerifyEmail`**: Requires email verification
- **HasApiTokens**: Supports Laravel Sanctum authentication
- **Notifiable**: Can receive notifications
- **Hidden Fields**: Password and tokens never serialized
- **Casts**: Email verification timestamp as datetime

### Relationships

- **One-to-Many**: User has many Bookings

### Usage Example

```php
// Get all bookings for authenticated user
$bookings = Auth::user()->bookings;

// Check if user is admin
if (Auth::user()->isAdmin) {
    // Admin functionality
}
```

## Tours Model

**Location**: `~/workspace/source/app/Models/Tours.php`

```php
class Tours extends Model
{
    use HasFactory;

    protected $fillable = [
        'hotel',
        'transport',
        'image',
        'per_person_sharing',
        'single_room',
        'meals'
    ];
}
```

### Database Table Fields

Based on migration (`2019_05_01_071637_create_tours_table.php`):

- `id` - Primary key
- `hotel` - Hotel/accommodation name
- `transport` - Transportation details
- `location` - Tour location
- `category` - Main category
- `sub_category` - Subcategory
- `image`, `image1`, `image2`, `image3`, `image4`, `image5`, `image6` - Multiple images
- `per_person_sharing` - Price for shared accommodation
- `single_room` - Price for single room
- `meals` - Meal plan details
- `additional_info` - Optional extra information
- `user_id` - Creator (admin) ID
- `published` - Publication status (0=draft, 1=published)
- `created_at`, `updated_at` - Timestamps

### Usage Examples

<CodeGroup>

```php Creating a Tour
$tour = new Tours();
$tour->hotel = 'Serengeti Safari Lodge';
$tour->transport = 'Land Cruiser 4x4';
$tour->location = 'Serengeti National Park';
$tour->category = 'Wildlife Safari';
$tour->sub_category = 'Big Five Safari';
$tour->per_person_sharing = '$1,500';
$tour->single_room = '$2,000';
$tour->meals = 'Full Board';
$tour->user_id = Auth::id();
$tour->save();
```

```php Querying Tours
// Get all published tours
$tours = Tours::where('published', 1)->get();

// Get tours by category
$safaris = Tours::where('category', 'Safari Tours')
    ->where('published', 1)
    ->paginate(20);

// Random tours for homepage
$featured = Tours::inRandomOrder()->limit(6)->get();
```

```php Publishing a Tour
$tour = Tours::find($id);
$tour->published = 1;
$tour->update();
```

</CodeGroup>

```php
class Booking extends Model

        'phone_number',
```

### Custom Table Name

Note: Model is named `Booking` but table is `tour_bookings` (explicitly defined).

### Complete Fields

From migration `2022_05_01_081632_create_tour_booking_table.php`:

- `transaction_code` - Payment reference
- `first_name`, `last_name` - Customer name
- `email`, `phone_number` - Contact info
- `country` - Customer country
- `arrival_date` - Tour start date
- `adults`, `kids` - Number of travelers
- `destination` - Booked tour
- `special_requirements` - Dietary/accessibility needs
- `user_id` - Booking owner
- `status` - Confirmation status (0=pending, 1=confirmed)

### Usage Example

```php
// Get user's bookings
$bookings = Booking::where('user_id', Auth::id())
    ->latest()
    ->paginate(10);

// Confirm booking
$booking = Booking::find($id);
$booking->status = 1;
$booking->update();
```

## Model Relationships

While most relationships are implicit through foreign keys, here are the defined relationships:

### Explicit Relationships

```php
// User Model (app/Models/User.php:45-48)
public function bookings()
{
    return $this->hasMany('App\Booking');
}
```

### Implicit Relationships

Based on database schema:

- **User → Tours**: `user_id` in tours table
- **User → Blogs**: `user_id` in blogs table
- **User → Products**: `user_id` in products table
- **User → Services**: `user_id` in services table
- **User → Events**: `user_id` in events table
- **User → Accommodations**: `user_id` in accomodations table

### Defining Missing Relationships

To improve code clarity, consider adding these to respective models:

```php
// In Tours model
public function creator()
{
    return $this->belongsTo(User::class, 'user_id');
}

// In User model
public function tours()
{
    return $this->hasMany(Tours::class);
}

public function blogs()
{
    return $this->hasMany(Blog::class);
}
```

## Mass Assignment Protection

### Fillable vs Guarded

Safaris Web uses the `$fillable` whitelist approach:

```php
// Only these fields can be mass-assigned
protected $fillable = ['name', 'email', 'password'];
```

<Warning>
**Security Note**: Always define `$fillable` or `$guarded` to prevent mass assignment vulnerabilities.
</Warning>

### Missing Fillable Definitions

These models need complete `$fillable` arrays:

- **Service**: No fillable defined
- **Highlight**: No fillable defined
- **Tours**: Missing many fields (location, category, etc.)
- **Category**: No fillable defined

## Common Eloquent Operations

### Querying

<CodeGroup>

```php Find by ID
$tour = Tours::find($id);
$tour = Tours::findOrFail($id); // Throws 404 if not found
```

```php Where Clauses
$tours = Tours::where('published', 1)->get();
$tours = Tours::where('category', 'Safari')
    ->where('published', 1)
    ->get();
```

```php Latest/Oldest
$blogs = Blog::latest()->get(); // Order by created_at DESC
$blogs = Blog::oldest()->get();
```

```php Pagination
$tours = Tours::paginate(20);
$tours = Tours::where('published', 1)->paginate(15);
```

</CodeGroup>

### Creating Records

<CodeGroup>

```php Create Method
$tour = Tours::create([
    'hotel' => 'Safari Lodge',
    'transport' => 'Land Cruiser',
    // ... other fields
]);
```

```php New Instance
$tour = new Tours();
$tour->hotel = 'Safari Lodge';
$tour->transport = 'Land Cruiser';
$tour->save();
```

</CodeGroup>

### Updating Records

<CodeGroup>

```php Update Method
$tour = Tours::find($id);
$tour->update([
    'published' => 1,
    'hotel' => 'Updated Name'
]);
```

```php Save Method
$tour = Tours::find($id);
$tour->published = 1;
$tour->save();
```

</CodeGroup>

### Deleting Records

```php
$tour = Tours::find($id);
$tour->delete();

// Or direct delete
Tours::destroy($id);
```

```php
{
    protected static function boot()
    {
        parent::boot();
        
        static::creating(function ($tour) {
            // Auto-set user_id
            $tour->user_id = auth()->id();
        });
        
        static::updating(function ($tour) {
            // Log changes
        });
    }
}
```
