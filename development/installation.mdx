## Prerequisites

Before installing Safaris Web, ensure your system meets these requirements:

- **PHP**: ^8.0.2 or higher
- **Composer**: Latest version
- **Node.js & NPM**: For frontend asset compilation
- **Database**: MySQL or PostgreSQL
- **Web Server**: Apache or Nginx

## Installation Steps

### 1. Clone the Repository

```bash
cd ~/workspace/source
git clone <repository-url> .
```

### 2. Install PHP Dependencies

Safaris Web uses Composer to manage PHP dependencies. Install all required packages:

```bash
composer install
```

#### Core Dependencies

The application includes the following key packages from `composer.json`:

**Production Dependencies:**
- `laravel/framework: ^9.2` - Laravel 9 framework core
- `anhskohbo/no-captcha: ^3.4` - Google reCAPTCHA integration
- `guzzlehttp/guzzle: ^7.2` - HTTP client for API requests
- `laravel/sanctum: ^2.14.1` - API authentication
- `laravel/tinker: ^2.7` - Interactive REPL for Laravel
- `laravel/ui: ^3.4` - Authentication scaffolding

**Development Dependencies:**
- `fakerphp/faker: ^1.9.1` - Generate fake data for testing
- `laravel/sail: ^1.0.1` - Docker development environment
- `phpunit/phpunit: ^9.5.10` - Testing framework
- `spatie/laravel-ignition: ^1.0` - Error page for Laravel

### 3. Install Frontend Dependencies

Install Node.js packages for frontend asset compilation:

```bash
npm install
```

#### Frontend Dependencies

From `package.json`, the application uses:

- `laravel-mix: ^6.0.6` - Asset compilation
- `bootstrap: ^5.1.3` - CSS framework
- `axios: ^0.30` - HTTP client for AJAX requests
- `sass: ^1.32.11` & `sass-loader: ^11.0.1` - SCSS compilation
- `@popperjs/core: ^2.10.2` - Tooltip & popover positioning

### 4. Environment Configuration

Copy the example environment file and generate an application key:

```bash
cp .env.example .env
php artisan key:generate
```

See the [Configuration](/development/configuration) page for detailed `.env` setup.

### 5. Storage Link

Create a symbolic link from `public/storage` to `storage/app/public` for file uploads:

```bash
php artisan storage:link
```

This is automatically configured in `composer.json` post-install hooks:

```json
"post-install-cmd": [
  "ln -sr storage/app/public public/storage"
]
```

### 6. Database Setup

Run migrations to create database tables:

```bash
php artisan migrate
```

Optionally, seed the database with sample data:

```bash
php artisan db:seed
```

See [Database Setup](/development/database-setup) for more details.

### 7. Compile Assets

<CodeGroup>

```bash Development
npm run dev
```

```bash Watch Mode
npm run watch
```

```bash Production
npm run prod
```

</CodeGroup>

## Running the Application

### Development Server

Start the Laravel development server:

```bash
php artisan serve
```

The application will be available at `http://localhost:8000`.

### Laravel Sail (Docker)

Alternatively, use Laravel Sail for a Docker-based development environment:

```bash
./vendor/bin/sail up
```

## Troubleshooting

### Permission Issues

If you encounter permission errors, set correct permissions:

```bash
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
```

### Composer Memory Limit

If Composer runs out of memory:

```bash
php -d memory_limit=-1 /usr/local/bin/composer install
```

### Clear Application Cache

```bash
php artisan cache:clear
php artisan config:clear
php artisan view:clear
```
